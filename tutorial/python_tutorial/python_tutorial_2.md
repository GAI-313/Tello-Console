## 四則演算 とは
　四則演算とは、皆さんが小学生で学んだ

- 足し算
- 引き算
- 掛け算
- 割り算

の **4つの基本的な計算方法** のことを指します。このドキュメントでは、**Python を使って四則演算を行う方法** について説明します。

## 四則演算をやってみよう
　これで準備が整いました。Python を使って四則演算をやってみましょう。まずは足し算からです。

<a id="2.1"></a>
### 足し算
　以下のコードは、足し算の $1+1$ をPython で記述したものとなります。
```python
1+1
```
　実際に CUI 常にこれを記述すると、このようになります。このように ```>>>``` の横に入力したコードが表示されます。
```
>>> 1+1
```
　これを書いたら、**エンターキー** を押しましょう、するとかいたコードが実行され、結果が返されます。
```
>>> 1+1
2
>>>
```
　他にも色々足し算を書いてみましょう。
```math
10 + 10
```
```math
19 + 20
```
```math
99999999 + 9999999999
```
　このようにほぼ直感的に書けるかと思います。

> **演算子**<br>
> ```python
> 1+1
> ```
> この ```+``` などの計算式に使用する符号のことを **演算子** と言います。以降はこの記号の総称を **演算子** と呼びます。

<a id="2.2"></a>
### 引き算
　以下のコードは、引き算の $1-1$ を Python で記述したものとなります。引き算の演算子は、```-``` です。
```python
1-1
```
　実際に CUI 常にこれを記述すると、このようになります。
```
>>> 1-1
```
　これを書いたら、**エンターキー** を押しましょう、するとかいたコードが実行され、結果が返されます。
```
>>> 1-1
0
>>>
```
　他にも色々引き算を書いてみましょう。
```math
20 - 3
```
```math
10 - 20
```
　もちろん。以上のコードの通り、**負の値** も記述可能です。
```python
>>> 10-20
-10
>>>
```
　負の値を使った計算は、**-** をつけるだけです。
```python
-10-5
```
　もちろん。負の値を使った足し算も記述可能です。
```python
-10+5
```
　ちょっともずかしい表現になるのですが、**負の値として使っている** ```-``` **は数字として扱われます。** 例えば、-5 と -8 を足し算する場合、このように記述できます。
```python
-5+-8
```
> **見やすいコードを書こう**<br>
> ここまで計算式を ```1+1``` のように書いていましたが、以下のように **演算子と数字の間に半角文スペースを開けて** 書くこともできます。
> ```python
> 1 + 1
> ```
> このように書くことで、負の値なのか演算子なのかを見分けて書くことができるようになります。
> ```python
> -5 + -8
> ```
> 実は以下のように数値と演算子の間にいくら空間があっても計算することが可能です。
> ```python
> >>> 1             +              1
> 2
> ```
> ですが **普通に見にくいので** お勧めはしません。

<a id="2.4"></a>
### 割り算
　以下のコードは、割り算の $6\div2$ を Python で記述したものとなります。引き算の演算子は、```/``` または ```//``` です。<br>
　割り算はこのように **2種類の演算子** があります。一つずつ見てみましょう。<br>
　演算子 ```/``` を使って 6÷ 2 を書くと、このようになります。
```python
>>> 6/2
3.0
>>>
```
　他にも $3\div2$ を書いてみると
```python
>>> 3/2
1.5
>>>
```
　と返されます。<br>
　演算子 ```//``` を使って $6\div2$ を書くと、このようになります。
```python
>>> 6//2
3
>>>
```
　他にも $3\div2$ を書いてみると
```python
>>> 3//2
1
>>>
```
　と返されます。<br>
　計算結果が若干異なることに気づきましたか？<br>
　演算子 ```/``` を使うと、小数点以下の計算も行います。演算子 ```//``` を使うと、整数のみの計算を行います。

> **整数と小数**<br>
> 　整数とは、1、2、20、-1、-10 などの **1づつ増える数字** のことを指します。<br>
> 　一方小数とは 1.1、-0.1、3.141592 などの **小数点を含む数字** のことを指します。<br>
> 　Python では整数と小数を **異なるデータとして扱います** 。この意味については **今後データ型セクションで学習するので** 今はあまり気にしなくていいです。<br>
> 　データ型セクションで ```/``` と ```//``` の活用方法と整数と少数の違いについてよく理解できるようになります！

### 計算の優先順位
　以下の式を Python で実行してみましょう。
```python
2 + 2 * 3 + 3
```
　すると結果がこのようになります。
```python
>>> 2+2*3+3
11
>>> 
```
　どのように計算されたかみてみると、このような手順を踏んでいます。

1. $2\times3$ が計算され、このようになる。 $2+6+3$
2. $2+6$ が計算される、このようになる $8+3$
3. $8+3$ が計算され、計算終了。 $11$

　この計算過程には、以下の優先順位があります。

1. （）がついた式を優先する（優先度1位）
1. 割り算、掛け算が最も湯煎して計算される（優先度2位）
2. 左から順番に計算を行う（優先度3位）

　これらを踏まえて、もう一度上記の計算過程を見てみましょう。<br>
　まず、 $2+2\times3+3$ をみると、中央に掛け算がありますよね？そのためまず優先的に掛け算が初めに計算されます。よって $2+6+3$ となります。次に、掛け算はもうありませんので、左側から計算を行います。よって $8+3$ と続いて、最後は $11$ で計算終了です。<br>
　次に以下の式を見てみましょう。
```math
10+4\div2\times6\div2-10
```
この計算過程を見てみましょう。さて、先ほど計算式に掛け算または割り算がある場合は、この2つが最も優先されて計算されると言いました。しかしこの式では **複数の掛け算と割り算** があります。この場合、どれが優先的に計算されるのでしょうか？<br>
　掛け算、割り算のみの計算式を抜いてみましょう。すると以下の式を抜き出すことができます。
```math
4\div2\times6\div2
```
この場合、先ほどの **左から順番に計算を行う** が適応されます。つまり初めに $4\div2$ が計算されます。よって以下の結果となります。
```math
2\times6\div2
```
次に $2\times6$ が計算され、
$12\div2$
となり、
$6$
となります。結果以下の式に収束します。
```math
10+6-10
```
あとは今までと同じ過程で計算され、結果は $6$ となります。実際に Python に記述して確かめてみましょう。<br>
　最後に括弧を使った計算を見てみましょう。先ほどの式に括弧を付け加えました。
```math
(10+4)\div2\times6\div(2-10)
```
この計算式では **括弧がついた計算式が最も優先される** が適応されます。この場合、以下の手順で計算が行われます。<br>
　まず左側の括弧内の式 $(10+4)$ が計算され、結果 $14\div2\times6\div(2-10)$ となります。次に末尾の括弧内の式 $(2-10)$ が計算され、
結果 $14\div2\times6\div-8$ となります。以降は先ほど学んだ計算手順によって計算されます。<br>

1. $14\div2\times6\div-8=7\times6\div-8$
2. $7\times6\div-8=42\div-8$
3. $42\div-8=-6$

　演算子 ```//``` を利用しているため、計算結果は全て整数となり、少数以下の計算は省略されます。
